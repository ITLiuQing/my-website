给作品集新增一个作品:$ARGUMENTS

严格按下面的流程做,并遵守项目 CLAUDE.md 里的所有设计约定。

## 第 0 步:弄清作品信息

需要三样信息:**作品名**、**一句话简介**(它解决什么问题)、**一个合适的 emoji**。
- 用户没给全的,能从作品名合理推断的就自己定(emoji 通常可以自己选);
- 如果连作品是干什么的都不知道,先用一句话问清楚,不要瞎编。
- 给详情页取一个简短的英文文件名(小写,单词间用连字符,如 `habit-tracker.html`)。

## 第 1 步:首页 index.html 加卡片

1. 先读 index.html,数一下现有 `cover-N` 到几号了,看清每个封面用的渐变色。
2. 新增一个 `cover-N+1` 样式:**必须是暖色系渐变**(日落色盘:杏橙/珊瑚/玫瑰/暖紫粉一路),且颜色不能和任何已有封面重复或过于接近;禁止冷色。
3. 在 `.cards` 里、"Claude Code 学习笔记"这张不带链接的卡片**之前**,插入新卡片,结构和现有三张可点击卡片完全一致:
   - `<a class="card" href="works/文件名.html">`
   - 封面色块 + emoji
   - `<h3>作品名</h3>` + 一句话简介
   - `<span class="badge">整理中</span>` + `<span class="card-more">查看详情 →</span>`

## 第 2 步:works/ 里生成详情页

1. 先完整读一个现有详情页(如 `works/title-helper.html`)当模板,新页面结构、样式必须和它一致:
   - `<title>作品名 · 林小满</title>` + 同款 🌅 favicon
   - 固定"← 返回首页"按钮(链接 `../index.html`)
   - 封面色块:渐变和首页新卡片**同一个**,emoji 也相同
   - 标题 + "整理中"徽章
   - 三段内容:**它解决什么问题 / 我怎么做出来的 / 用下来的感受**——根据作品信息写,语气参考现有页面(第一人称、口语、真诚),没有的信息宁可写短也不要编造具体数据
   - 页脚署名(微信 + 邮箱,含 `footer span` 不换行样式)
2. 确认双向链接:首页卡片 → 详情页,详情页"返回首页" → 首页。

## 第 3 步:手机窄屏自查(必做)

在项目目录起一个临时本地服务器(浏览器不允许直接开 file:// 页面),用 Playwright 打开:

1. 桌面宽度看一遍首页新卡片和新详情页;
2. 375px 宽下再看一遍:新卡片单列显示是否正常、详情页文字有没有溢出(用脚本查 `scrollWidth > clientWidth` 和越界元素);
3. 点击新卡片确认真的能跳转、详情页"返回首页"能跳回;
4. 查完关掉临时服务器和浏览器。

发现问题当场修掉再复查。

## 第 4 步:汇报

最后**只用一句话**汇报:加了什么作品、卡片和详情页都好了、手机自查通过没有。不要长篇报告。
(提醒:改动只在本地,要用户确认后才 commit/push 上线。)
